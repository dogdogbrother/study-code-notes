# jest 测试

前端的jest测试和正常的职业测试是不一样的，需要对js熟悉是一方面，主要的是要对前端代码和功能比较熟悉才行。

基本可以说jest模式的测试只能前端来做。

## 易用性
虽然很多前端工具里面都有jest的测试用例，但是我相信很多前端都没怎么用过jest，主要是因为jest的功能比较单一但是api却不少。

如果想配合 vue 或者 react 等框架，还要有对应的 jest 工具，想要使用就还要学习一番。

并且编写测试用例着实浪费时间，并且很难hold的住复杂的业务代码。

## 作用
其实对于开发是没啥用的，能提升的那点代码质量的性价比远远比不上耗费的时间，当业务有改动的时候，不仅要改代码，还要改测试用例。。。

我个人认为jest这种前端测试的作用是在于多人协作以及代码回归。

例如我改一个人的组件，但是我不知道会不会因为我的改动造成他的代码bug，如果他有核心函数的测试用例，我就可以通过跑这个测试用例来判断我的代码是否影响了原来代码。

代码回归也是相同意思。

## 重要性
浏览了一遍慕课网 dell lee 的jest课程，实在不觉得jest是前端必学的。。。

现在是2020年2月，如果是TS对前端重要性是10分的话，我觉得jest也就是3分吧。。。

可学可不学，没啥意思。
